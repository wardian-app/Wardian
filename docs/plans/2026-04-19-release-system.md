# Release System Implementation Plan

> **For agentic workers:** Execute steps in order. Use checkbox (`- [ ]`) syntax for tracking. Commit after each task unless the task says otherwise. Do not skip verification steps — this plan has few unit-testable parts, so verification is mostly `git status`, schema checks, and CI dry-runs.

**Goal:** Ship a multi-OS GitHub Release pipeline for Wardian with automated version bumps and changelog generation, per `docs/specs/021-release-system.md`.

**Architecture:** `release-please` maintains a "Release PR" on `main` that batches Conventional Commits into version + CHANGELOG bumps. Merging the PR tags `v{version}`, which triggers `.github/workflows/release.yml` to build NSIS + DMG + AppImage + DEB bundles via `tauri-apps/tauri-action` and publish them to a GitHub Release.

**Tech Stack:** GitHub Actions, `googleapis/release-please-action@v4`, `tauri-apps/tauri-action@v0`, Tauri 2 bundler, Node 20, Rust stable.

**Branch:** `feat/release-system` (already created from `main`).

## Task 3: Add release-please config

**Files:**
- Create: `release-please-config.json` (repo root)

- [ ] **Step 1: Create the file**

Write exactly:

```json
{
  "$schema": "https://raw.githubusercontent.com/googleapis/release-please/main/schemas/config.json",
  "release-type": "node",
  "bump-minor-pre-major": true,
  "bump-patch-for-minor-pre-major": false,
  "include-v-in-tag": true,
  "packages": {
    ".": {
      "package-name": "wardian",
      "changelog-path": "CHANGELOG.md",
      "changelog-sections": [
        { "type": "feat", "section": "Features" },
        { "type": "fix", "section": "Bug Fixes" },
        { "type": "perf", "section": "Performance" },
        { "type": "docs", "section": "Documentation", "hidden": false },
        { "type": "refactor", "section": "Refactoring", "hidden": true },
        { "type": "test", "section": "Tests", "hidden": true },
        { "type": "ci", "section": "CI/CD", "hidden": true },
        { "type": "chore", "section": "Miscellaneous", "hidden": true }
      ],
      "extra-files": [
        {
          "type": "json",
          "path": "src-tauri/tauri.conf.json",
          "jsonpath": "$.version"
        },
        {
          "type": "toml",
          "path": "src-tauri/Cargo.toml",
          "jsonpath": "$.package.version"
        }
      ]
    }
  }
}
```

Key choices (documented so you can explain them if questioned):
- `release-type: node` — tells release-please to treat `package.json` as the primary version file.
- `bump-minor-pre-major: true` — while `<1.0.0`, breaking changes become minor bumps instead of major. Keeps the project in `0.x` until you explicitly decide to ship `1.0`.
- `bump-patch-for-minor-pre-major: false` — `feat:` commits still bump minor (`0.2.1 → 0.3.0`), not patch. Use `fix:` for patches.
- `include-v-in-tag: true` — produces `v0.3.0` tags, which `.github/workflows/release.yml` will match with `v*`.
- `extra-files` — keeps `tauri.conf.json` and `Cargo.toml` in lockstep with `package.json`.
- Conventional types `refactor`, `test`, `ci`, `chore` are hidden from CHANGELOG but still trigger a release when they land on `main` as the only change? No — none of them trigger a version bump; only `feat`, `fix`, `perf`, or `BREAKING CHANGE` do. A release PR only opens when at least one bump-triggering commit exists.

## Task 5: Pin explicit Tauri bundle targets

**Files:**
- Modify: `src-tauri/tauri.conf.json` (`bundle.targets` field)

Current state: `"targets": "all"` produces MSI + NSIS on Windows and DEB + AppImage + RPM on Linux. The spec excludes MSI and RPM.

- [ ] **Step 1: Read the current file**

Use the Read tool on `src-tauri/tauri.conf.json`. Note the exact formatting of the `bundle` object.

- [ ] **Step 2: Replace the `targets` field**

Using the Edit tool, change:

```json
    "active": true,
    "targets": "all",
```

to:

```json
    "active": true,
    "targets": ["nsis", "app", "dmg", "deb", "appimage"],
```

Explanation of each target:
- `nsis` — Windows installer `.exe` (skip `msi`).
- `app` — macOS `.app` bundle (input to `dmg`; Tauri builds it anyway but listing it explicitly is clearer).
- `dmg` — macOS disk image.
- `deb` — Debian/Ubuntu package.
- `appimage` — portable Linux binary.

Intentionally excluded: `msi` (Windows), `rpm` (Linux), `updater` (deferred to signing spec).

## Task 6: Add the build & publish workflow

**Files:**
- Create: `.github/workflows/release.yml`

This is the largest file in the plan. Write it exactly as shown; do not improvise.

- [ ] **Step 1: Create the file**

Write exactly:

```yaml
name: Release

on:
  push:
    tags:
      - "v*"
  workflow_dispatch:
    inputs:
      dry_run:
        description: "Dry run (build artifacts, do not create a release)"
        required: false
        default: "true"

permissions:
  contents: write

jobs:
  create-release:
    name: Create draft release
    if: github.event_name == 'push'
    runs-on: ubuntu-latest
    outputs:
      release_id: ${{ steps.create.outputs.result }}
      is_prerelease: ${{ steps.detect.outputs.prerelease }}
    steps:
      - uses: actions/checkout@v4
      - name: Detect prerelease
        id: detect
        run: |
          TAG="${GITHUB_REF#refs/tags/}"
          if [[ "$TAG" == *-preview.* || "$TAG" == *-beta.* || "$TAG" == *-rc.* ]]; then
            echo "prerelease=true" >> "$GITHUB_OUTPUT"
          else
            echo "prerelease=false" >> "$GITHUB_OUTPUT"
          fi
      - name: Extract changelog entry
        id: changelog
        run: |
          TAG="${GITHUB_REF#refs/tags/}"
          VERSION="${TAG#v}"
          awk -v ver="$VERSION" '
            $0 ~ "^## \\[" ver "\\]" { capture = 1; next }
            capture && /^## \[/ { exit }
            capture { print }
          ' CHANGELOG.md > release_notes.md
          if [ ! -s release_notes.md ]; then
            echo "Release notes for $VERSION not found in CHANGELOG.md" >&2
            echo "See CHANGELOG.md" > release_notes.md
          fi
      - name: Create draft release
        id: create
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const body = fs.readFileSync('release_notes.md', 'utf8');
            const tag = context.ref.replace('refs/tags/', '');
            const res = await github.rest.repos.createRelease({
              owner: context.repo.owner,
              repo: context.repo.repo,
              tag_name: tag,
              name: `Wardian ${tag}`,
              body,
              draft: true,
              prerelease: ${{ steps.detect.outputs.prerelease }},
            });
            return res.data.id;
          result-encoding: string

  build:
    name: Build (${{ matrix.platform.os }}${{ matrix.platform.arch && format(' / {0}', matrix.platform.arch) || '' }})
    needs: [create-release]
    if: always() && (needs.create-release.result == 'success' || github.event_name == 'workflow_dispatch')
    strategy:
      fail-fast: false
      matrix:
        platform:
          - os: windows-latest
            name: windows
          - os: macos-latest
            arch: aarch64
            name: macos-arm64
            args: --target aarch64-apple-darwin
            rust-target: aarch64-apple-darwin
          - os: macos-13
            arch: x86_64
            name: macos-x64
            args: --target x86_64-apple-darwin
            rust-target: x86_64-apple-darwin
          - os: ubuntu-22.04
            name: linux
    runs-on: ${{ matrix.platform.os }}
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: "npm"

      - name: Install Rust
        uses: dtolnay/rust-toolchain@stable
        with:
          targets: ${{ matrix.platform.rust-target || '' }}

      - name: Rust Cache
        uses: Swatinem/rust-cache@v2
        with:
          workspaces: src-tauri

      - name: Install Linux deps
        if: matrix.platform.os == 'ubuntu-22.04'
        run: |
          sudo apt-get update
          sudo apt-get install -y libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev patchelf libayatana-appindicator3-dev

      - name: Install npm dependencies
        run: npm ci

      - name: Build and upload (tag push)
        if: github.event_name == 'push'
        uses: tauri-apps/tauri-action@v0
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          releaseId: ${{ needs.create-release.outputs.release_id }}
          args: ${{ matrix.platform.args || '' }}

      - name: Build (dry run)
        if: github.event_name == 'workflow_dispatch'
        uses: tauri-apps/tauri-action@v0
        with:
          args: ${{ matrix.platform.args || '' }}

      - name: Upload dry-run artifacts
        if: github.event_name == 'workflow_dispatch'
        uses: actions/upload-artifact@v4
        with:
          name: wardian-${{ matrix.platform.name }}
          path: |
            src-tauri/target/release/bundle/**/*.exe
            src-tauri/target/release/bundle/**/*.dmg
            src-tauri/target/release/bundle/**/*.AppImage
            src-tauri/target/release/bundle/**/*.deb
            src-tauri/target/*/release/bundle/**/*.exe
            src-tauri/target/*/release/bundle/**/*.dmg
            src-tauri/target/*/release/bundle/**/*.AppImage
            src-tauri/target/*/release/bundle/**/*.deb
          if-no-files-found: warn

  publish:
    name: Publish release
    needs: [create-release, build]
    if: github.event_name == 'push' && needs.build.result == 'success'
    runs-on: ubuntu-latest
    steps:
      - name: Publish the draft release
        uses: actions/github-script@v7
        with:
          script: |
            await github.rest.repos.updateRelease({
              owner: context.repo.owner,
              repo: context.repo.repo,
              release_id: ${{ needs.create-release.outputs.release_id }},
              draft: false,
            });
```

Notes for the executor:
- Heredoc-style awk script in `Extract changelog entry` must be preserved exactly — it parses CHANGELOG.md between the `## [X.Y.Z]` header and the next `## [` header.
- `tauri-action` uploads to the draft release automatically when `releaseId` is set.
- The dry-run path uses `actions/upload-artifact@v4` instead — artifacts appear under the workflow run, not a release.
- `fail-fast: false` so one platform's failure doesn't cancel the others mid-build.

## Out of Scope (Do Not Attempt)

- Code signing configuration (Windows EV, Apple Developer, notarization).
- `tauri-plugin-updater` integration.
- OS package manager submissions (winget, Scoop, AUR, Homebrew Cask, Flathub).
- The `wardian-cli` crate or its distribution wrappers.

If any of these come up as "wouldn't it be easy to also...", STOP and surface to the human.

## Failure Modes to Watch For

1. **release-please opens no PR after merging to main** — likely no `feat:`/`fix:`/`perf:` commits since last tag. Land one and re-check.
2. **`cargo check` fails after Task 5** — the `targets` array syntax may be wrong for the Tauri version in use. Check `src-tauri/Cargo.toml` for the `tauri` dependency version and consult that version's config schema.
3. **`tauri-action` build fails on macOS x86_64** — the `macos-13` runner's availability changes periodically. If deprecated, pin to `macos-14` with `--target x86_64-apple-darwin` (still builds x86_64 via cross-compile).
4. **CHANGELOG awk script produces empty release notes** — the `## [X.Y.Z]` header format must match exactly. Verify CHANGELOG.md formatting; fix the regex or CHANGELOG, don't silently swallow.
5. **Release PR body references wrong repo URL** — if `git remote get-url origin` doesn't match `tgemicioglu/wardian`, update the README link in Task 7.
