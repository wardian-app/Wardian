## Design Context

### Users

Developers and AI orchestrators who need a high-performance, persistent environment ("Integrated Agent Environment") to manage multiple long-running autonomous agent sessions, workflows, and global context.

### Brand Personality

Tactile, high-tech, ecological. The interface should evoke an emotion of being Technical & Powerful.

### Aesthetic Direction

A mix of modern IDEs, powerful knowledge graphs, and professional design tools.

- **References**: Obsidian (tabbing, minimal interface), VSCode (layout, command palette, terminal integration), Figma (canvas, node-builder).
- **Anti-references**: Vim (too obtuse/CLI-only), Termux (barebones terminal), ChatGPT (generic chat interface).
- **Theme**: **Light mode is the default**. The system must support robust Dark mode, Color-blind, and High-contrast versions.

### Design Principles

1. **Direct Visual Manipulation**: Prioritize interactive, tactile controls (drag-and-drop, node-linking, toggleable sidebars) as the primary mode of interaction.
2. **Optional Keyboard Sovereignty**: Support power users through an optional but robust command palette and keyboard-based navigation.
3. **High Signal-to-Noise**: Maximize information density for technical telemetry while ensuring absolute clarity and legibility.
4. **Accessible Inclusivity**: Ensure high-contrast and color-blind friendly themes are first-class citizens in the design system.
5. **Habitat Isolation**: Visually reinforce the "terrarium/habitat" metaphor through structured, clearly bounded components (cards, nodes, panes) that maintain connection without clutter.
